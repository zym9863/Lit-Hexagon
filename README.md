# 旋转六边形弹球游戏

一个基于 Lit 框架和 Canvas 2D 的物理仿真游戏，展示了一个在旋转六边形内弹跳的小球，具有真实的重力、摩擦力和碰撞反弹效果。

## 🎮 游戏特性

- **真实物理模拟**：小球受重力影响自由下落，具有速度衰减和能量损耗
- **旋转墙面交互**：六边形持续旋转，墙面的运动会影响球的反弹方向和速度
- **精确碰撞检测**：实现了球与旋转六边形墙面的精确碰撞检测和响应
- **视觉效果增强**：
  - 运动轨迹显示
  - 碰撞闪光效果
  - 渐变色彩和发光效果
  - 高DPI屏幕适配

## 🚀 快速开始

### 前置要求

- Node.js 18.0 或更高版本
- pnpm 8.0 或更高版本

### 安装依赖

```bash

```


然后在浏览器中打开 http://localhost:5173

### 构建生产版本

```bash
pnpm build
```

### 预览生产版本

```bash
pnpm preview
```

## 🏗️ 技术架构

### 核心技术栈

- **Lit 3.3**: Web Components 框架，负责组件生命周期管理
- **TypeScript 5.8**: 类型安全的 JavaScript 超集
- **Vite 7.1**: 现代前端构建工具
- **Canvas 2D API**: 游戏渲染引擎

### 项目结构

```
src/
├── components/
│   └── hex-bounce-game.ts    # 主游戏组件
├── physics/
│   ├── vector2.ts            # 2D向量数学工具
│   ├── hex.ts                # 六边形几何定义
│   ├── collision.ts          # 碰撞检测与响应
│   └── engine.ts             # 物理引擎主循环
├── my-element.ts             # 应用入口组件
└── index.css                 # 全局样式
```

### 物理引擎特性

#### 固定时间步进
- 采用 120Hz 的固定物理更新频率
- 使用时间累加器处理变帧率
- 最大帧时间限制防止螺旋死亡

#### 碰撞系统
- 六边形使用内切圆半径（apothem）定义
- 世界空间法线实时计算
- 多面碰撞迭代处理
- 速度阈值防止数值抖动

#### 物理参数

| 参数 | 默认值 | 说明 |
|-----|--------|-----|
| gravity | 1400 px/s² | 重力加速度 |
| frictionGamma | 0.8 /s | 全局速度衰减系数 |
| restitution | 0.9 | 法向恢复系数（弹性） |
| tangentDamping | 0.98 | 切向速度阻尼 |
| rotationSpeed | 0.6 rad/s | 六边形旋转速度 |
| ballRadius | apothem × 0.07 | 球半径比例 |
| maxSpeed | 2000 px/s | 最大速度限制 |

```typescript
  rotationSpeed="0.8">

```

### 性能优化

- 向量对象复用减少 GC 压力
- 固定时间步保证物理稳定性
- Canvas 层级优化减少重绘
- RequestAnimationFrame 优化渲染

## 📝 开发说明

### 向量运算优化
所有向量运算都支持结果对象复用，避免频繁创建临时对象：

```typescript
// 复用 result 对象
add(v1, v2, result);  

// 而不是每次创建新对象
const result = add(v1, v2);
```

### 碰撞响应流程
1. 检测所有边的穿透深度
2. 选择最大穿透的边作为碰撞面
3. 计算墙面在接触点的速度
4. 分解相对速度为法向和切向分量
5. 应用恢复系数和阻尼
6. 合成新的速度向量

### 渲染优化
- 使用 devicePixelRatio 适配高清屏
- 轨迹使用 alpha 衰减实现淡出效果
- 碰撞闪光使用指数衰减

## 🐛 已知问题与限制

- 极高速度可能导致穿透（已通过速度限制缓解）
- 多面同时碰撞时可能有轻微抖动（已通过速度阈值处理）
- 在极低帧率设备上物理表现可能不稳定
